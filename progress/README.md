# progress/ — memoria externa

Esta carpeta es la memoria durable del proyecto. La conversación no lo es.

## Cuándo escribir

- En cada avance importante: una decisión tomada, un paso terminado, un obstáculo.
- Siempre antes de cerrar una sesión con trabajo a medias (nota de continuación).

## Formato

Un archivo por tarea, llamado `AAAA-MM-DD-nombre-corto.md`:

```markdown
# <nombre de la tarea>
Estado: en curso | trabado | terminado

## Objetivo
Qué significa "terminado" para esta tarea, en una o dos líneas.

## Hecho hasta ahora
- [x] paso — resultado, qué se produjo

## Decisiones
- Elegí X en vez de Y porque…

## Próximos pasos
- La siguiente acción concreta.

## Obstáculos
-
```

## La prueba de una buena nota

Alguien que no vio nada de la conversación tiene que poder retomar la tarea
leyendo solo este archivo. Si no puede, la nota está incompleta.
